# Documentation

This directory contains only current, authoritative project documentation. Historical planning notes and obsolete deployment records are intentionally removed; Git history retains them. Detailed current feature contracts live in `docs/features/`.

## Read By Task

| Before changing... | Read |
| --- | --- |
| Anything | [Project Rules](project-rules.md) |
| Question generation, rank difficulty, Daily content | [Curriculum](curriculum.md) |
| Daily, Ear Training, Sound Gates, XP, ranks, quests, hints | [Practice and Progression](practice-and-progression.md) |
| Piano, guitar, scales, chords, songwriting, settings | [Instruments and Creation](instruments-and-creation.md) |
| Product purpose, audience, visual direction | [Product](PRODUCT.md) and [Design System](DESIGN.md) |
| Living Music City runtime, districts, assets, privacy, and phase gates | [Living Music City](features/living-music-city.md) and [Implementation Specification](../layout-update.md) |
| Recreate the repository, route/API/model map | [Architecture and Reproduction](architecture.md) |
| Detailed feature behavior and ownership | [Feature Contracts](features/README.md) |
| Deployment | [Deployment](fly-deployment.md) |
| Security | [Security](security.md) |

## Authority Order

1. [Project Rules](project-rules.md)
2. [Curriculum](curriculum.md)
3. [Practice and Progression](practice-and-progression.md) or [Instruments and Creation](instruments-and-creation.md)
4. [Architecture](architecture.md) and [Feature Contracts](features/README.md)
5. [Product](PRODUCT.md) and [Design System](DESIGN.md)
6. Deployment and security operational guides

When documents conflict, follow the higher document and update the lower one in the same change.
