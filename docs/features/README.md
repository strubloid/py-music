## Required Feature Sections

Every new or substantially changed feature contract must state: purpose and learner outcome; routes and entry points; user flow; typed data/API contract; frontend/backend ownership; persistence; authority/security boundary; accessibility; dependencies; tests; rollout/migration; known boundaries; and documentation links.
